# raspicat-vla

ROS2 Humble nodes for running Vision-Language-Action (VLA) navigation models on
the Raspberry Pi Cat (rt-net `raspicat`).

The repository defines a model-agnostic edge / remote split: the lightweight
edge runs on the robot, a remote workstation hosts the heavy VLA policy, and a
gRPC stream carries observations one way and action embeddings the other. The
same interface supports multiple backends — the dummy server (for CI / MVP),
[AsyncVLA](https://asyncvla.github.io/), and
[OmniVLA](https://omnivla-nav.github.io/).

## Workspace layout

This repository is itself a colcon workspace.

```
src/raspicat_vla_msgs/      # ROS2 messages, services, actions (model-agnostic)
src/raspicat_vla_proto/     # gRPC python stubs + ROS2 ⇄ proto conversion helpers
src/raspicat_vla_remote/    # gRPC server: dummy backend + (planned) AsyncVLA / OmniVLA
src/raspicat_vla_edge/      # Edge ROS2 nodes (lifecycle, path follower)
src/raspicat_vla_bringup/   # Launch composition
```

External submodules under `external/` (not built by colcon):

```
external/AsyncVLA/          # upstream AsyncVLA reference implementation
external/OmniVLA/           # upstream OmniVLA reference implementation
external/raspicat-sim-docker
```

## gRPC interface

`proto/raspicat_vla.proto` defines the model-agnostic `raspicat_vla.v1.VLAService`:

- `StreamInfer(stream Observation) returns (stream ActionEmbedding)`
- `GetModelInfo(ModelInfoRequest) returns (ModelInfo)`

Stubs are regenerated by `scripts/gen_proto.sh` into
`src/raspicat_vla_proto/raspicat_vla_proto/`.

## Build

```bash
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install
source install/setup.bash
```

## Quick start (Plan-1 MVP)

Bring up the dummy server, the edge lifecycle node, and the path follower in
one launch file:

```bash
ros2 launch raspicat_vla_bringup mvp_local.launch.py
```

Inject a fake camera frame and goal so something flows through the pipeline:

```bash
python3 tools/publish_fake_image.py
```
